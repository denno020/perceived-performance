    Well we could have a statically built site that accepts user interaction

    Being statically built will go a long way to acing the Lighthouse audit, but that's only for cold visits.

    Once a user has loaded their first web page, and starts to interact with the site, a lot of that performance work is less relevant

    In an SPA, after that first render, where a site feels fast or not is mostly dependent on the users network connection, and the load on the server

    If the connection is slow, API calls will take longer to resolve

    Similarly, if the server is under high stress, it will take longer to process those requests

    Both of these factors are going to give the user an impression of a slow website, regardless of how close we get Lighthouse to 100 for the performance metric

    Now, we can control what we can control, like optimising our servers and adding caching layers, but the response time is never going to be instant
